## §1 護欄規則

### 寫入邊界（系統層強制）

| 規則 | 說明 |
|------|------|
| **唯一入口** | 只有 `lc apply` 可寫入 `canonical/`、`derived/` |
| **繞過偵測** | 直接編輯資料檔 → `lc doctor` hard fail |
| **變更流程** | 所有變更先進 `proposals/`，`--confirm` 才能 apply |
| **Provenance** | 每次寫入含 `operation_id/actor/timestamp/prompt_hash/model_version` |

### Claude CLI 護欄

| 允許 | 禁止 |
|------|------|
| ✅ 生成草稿到 `proposals/` | ❌ 直接修改 `raw/` `canonical/` |
| ✅ 分析建議、搜尋 Wiki | ❌ 呼叫外部指令/腳本 |
| ✅ dry-run 輸出 diff | ❌ 觸發自動化 pipeline |

### 資料外洩防護

**絕不可出**（硬性 redaction）:
- 姓名、電話、Email、地址、身分證件、生日
- 帳號/卡號/IBAN、銀行代碼、API key/Token
- 交易對手可識別資訊、memo/description 原文

**可出**（仍需最小化）:
- 按月彙總的收入/支出區間
- 分類占比、情境參數範圍

**流程**: 外送顯示 redacted 預覽 + `--confirm` + 只存 `export_hash`

### 容錯優先序

#### Hard Fail（立即停止）

- `schema_version`/`calc_version` 不相容且無 migration
- canonical 不可變約束破壞（raw 參照缺失、stable_id 重複且衝突）
- proposal 缺少必要 provenance 或校驗失敗
- 時間序列邏輯不可能（資產事件在資產建立前）

#### Soft Warning（允許繼續，標記風險）

- 去重窗口命中但相似度不足，需人工裁決
- 少量欄位缺失但可推導（幣別用帳戶預設）
- derived 與 canonical 不一致（提示重算）

#### Auto-Fix（可逆且審計）

- 格式正規化（日期/幣別/小數精度）
- 欄位別名映射、分類映射
- derived 重算、索引重建、快取失效
- **必須產生 `operation_id` 與可回滾 diff**

## §2 系統架構

### V2 六層架構

```
┌─────────────────────────────────────────────────────────────────┐
│  PRESENTATION │ CLI 輸出 / Markdown Wiki / AI 顧問介面          │
└─────────────────────────────────────────────────────────────────┘
                               ↑
┌─────────────────────────────────────────────────────────────────┐
│   GENERATION  │ 報表生成 / 指南生成 / Wiki 編譯                  │
└─────────────────────────────────────────────────────────────────┘
                               ↑
┌─────────────────────────────────────────────────────────────────┐
│   CALCULATE   │ lifetime / budget / projection / scenario       │
│               │ [現有 calculators/ 擴展]                         │
└─────────────────────────────────────────────────────────────────┘
                               ↑
┌─────────────────────────────────────────────────────────────────┐
│     DATA      │ 結構化數據 (YAML/CSV) + 索引 (對話/筆記)         │
│               │ [現有 models/ + io/ 擴展]                        │
└─────────────────────────────────────────────────────────────────┘
                               ↑
┌─────────────────────────────────────────────────────────────────┐
│     PARSE     │ 自然語言→結構化 / 支出推論 / 實體抽取            │
└─────────────────────────────────────────────────────────────────┘
                               ↑
┌─────────────────────────────────────────────────────────────────┐
│    CAPTURE    │ Claude Code 對話 / 筆記檔案 / 零散輸入           │
└─────────────────────────────────────────────────────────────────┘
```

### 資料流

```
用戶輸入 (對話/筆記/零散支出描述)
    ↓ CAPTURE
proposals/ (待確認項目)
    ↓ PARSE (Claude Code 輔助)
結構化數據 (expenses, decisions, research)
    ↓ DATA (三層結構)
raw/ (不可變) → canonical/ (可查詢) → derived/ (可重算)
    ↓ CALCULATE
projection, scenario 計算結果
    ↓ GENERATION
Wiki 頁面 / 報表 / 指南
    ↓ PRESENTATION
CLI 輸出 / Markdown 檔案 / AI 顧問回應
```

#### Phase 5 Stage 3 新增模組對應

| 層級 | 新增模組 | 說明 |
|------|----------|------|
| GENERATION | `decision_wiki.py`, `risk_matrix.py`, `sensitivity_report.py` | 決策報告生成 |
| CALCULATE | `advisor/risk_assessor.py`, `advisor/sensitivity_analyzer.py` | 風險與敏感度分析 |
| DATA | `io/advisor_derived_handler.py`, `models/provenance.py` | 衍生物追蹤 |

### 資料三層結構

```
~/.life-capital/
├── raw/              # 不可變原始輸入
│   ├── imports/      # CSV/銀行匯入，含 source_id + import_time + parser_version
│   └── manual/       # 手動記錄
├── canonical/        # 正規化結構化資料
│   ├── expenses/     # 按月份切檔: 2024-12.jsonl
│   ├── annotations/  # 人工註解拆分: 2024-12.jsonl
│   └── .version      # schema_version
├── derived/          # AI 推論/計算結果（可重算）
│   ├── reports/      # 月度現金流、預測
│   └── scenarios/    # 情境分析結果
└── proposals/        # 待確認變更（AI 產出落地點）
    └── pending/      # operation_id + actor + timestamp
```

### canonical 欄位設計

| 欄位 | 說明 |
|------|------|
| `stable_id` | UUID，一旦創建永不改 |
| `dedupe_key` | versioned hash，允許策略更新 |
| `occurred_at` | 消費日（優先） |
| `posted_at` | 入帳日 |
| `currency` | 幣別（預留跨幣） |
| `is_transfer` | 轉帳標記 |
| `reversal_of` | 退款/沖正關聯 |
| `source_row_ref` | 指回 raw (source_id + row_index + raw_hash) |

### 跨層能力

| 能力 | 說明 | 實作方式 |
|------|------|----------|
| **Audit** | 每筆資料來源追溯 | `source_id`, `created_by`, `created_at`, `operation_id` |
| **Version** | Schema/計算版本標註 | `schema_version`, `calc_version`, `migration_log` |
| **Access** | 雙人權限控制 | `owner`, `shared_with`, 2-of-2 高風險操作 |

## §5 Financial Memory 與雙人模型

### Financial Memory（決策記憶）

**定義**: 系統的「決策記憶」，記錄財務決策的完整脈絡與可追溯性。

#### 組成元素

| 元素 | 說明 | 儲存位置 |
|------|------|----------|
| **decisions.yaml** | 財務決策記錄（買房/投資/目標調整） | `canonical/decisions/` |
| **operation_id** | 每次變更的唯一識別碼 | 每筆 proposal 與 canonical 記錄 |
| **migration_log** | Schema 版本變更歷程 | `canonical/.migrations/` |
| **scenario_assumptions** | 情境分析的假設參數快照 | `derived/scenarios/` |
| **provenance** | 變更來源追溯鏈 | `actor + timestamp + prompt_hash` |

#### 設計理念

- 「為什麼這樣做」比「怎麼做」更重要
- 決策記錄是給 6 個月後的自己看的
- AI 顧問可讀取 Financial Memory 作為上下文

### 雙人信任模型

#### 權限分級

| 類型 | 操作 | 確認方式 |
|------|------|----------|
| **日常** | 新增 raw、建立 proposal、重算 derived | 各自獨立 |
| **高風險** | schema 升級、migration、批次去重、刪除 canonical | 2-of-2 |
| **緊急** | 凍結寫入、回滾、停用資料來源 | 單人可做，事後追認 |

#### 衝突處理

- 同一 `stable_id` 競爭修改 → 進 proposal 審核佇列
- 超時未決 → 進入「暫停合併」狀態，不自動選邊

所有新路徑/檔名常數加入 `io/registry.py`：
```python

RESEARCH_DIR = "research"
WIKI_DIR = "wiki"
```

### 雙人共享考量

- Wiki 使用 Markdown 格式（Git 友好）
- 決策記錄包含 `participants` 欄位
- 支援不同視角的報表（個人/共同）

### Claude Code 整合

- Parse 層透過 subprocess 或 Skill 呼叫 Claude
- 不直接使用 API（利用現有訂閱）
- 提供 `--dry-run` 模式檢視 AI 推論結果

### 長期維護注意事項

#### 高技債風險點

- 多版本（schema_version + calc_version）需「版本相容矩陣」
- 去重規則過度聰明會讓人工裁決成本爆炸

#### 建議簡化

- migration 只允許「向前」；複雜變更新增欄位而非改既有
- derived 一律可刪可重算；不在 derived 存人工手改的真相
